# 詳細な使い方

## オプション詳細

### `<INPUT>` -- 入力パス（必須）

処理対象のファイルまたはディレクトリを指定します。

```bash
# 単一ファイル
automosaic photo.jpg

# ディレクトリ（直下のファイルのみ）
automosaic ./photos/

# ディレクトリ（再帰的に処理）
automosaic ./photos/ --recursive
```

### `--output` / `-o` -- 出力先パス

処理結果の保存先を指定します。デフォルトは `./output/` です。

```bash
automosaic photo.jpg -o ./processed/
```

ディレクトリが存在しない場合は自動的に作成されます。

### `--style` / `-s` -- モザイクスタイル

| 値 | 説明 |
|----|------|
| `pixelate` | ピクセル化（デフォルト）。対象領域を粗いブロックで表現 |
| `blur` | ガウシアンぼかし。対象領域をぼかして表現 |

```bash
automosaic photo.jpg --style blur
```

### `--strength` / `-S` -- モザイク強度

モザイクの強さを 1〜50 の整数で指定します（デフォルト: 15）。

- `pixelate`: 値が大きいほどブロックサイズが大きくなる
- `blur`: 値が大きいほどぼかしが強くなる

```bash
# 弱めのモザイク
automosaic photo.jpg --strength 5

# 強めのモザイク
automosaic photo.jpg --strength 40
```

### `--detector` / `-d` -- 検出エンジン

| 値 | 説明 | 必要パッケージ |
|----|------|---------------|
| `mediapipe` | 軽量・高速（デフォルト） | 標準インストールに含まれる |
| `yolo` | 高精度。小さい顔・横顔に強い | `pip install "automosaic[yolo]"` |

```bash
automosaic photo.jpg --detector yolo
```

### `--confidence` / `-c` -- 検出信頼度

顔検出の信頼度閾値を 0.0〜1.0 で指定します（デフォルト: 0.5）。

- 値を下げると検出漏れが減るが、誤検出が増える
- 値を上げると誤検出が減るが、検出漏れが増える

```bash
# 感度を上げる（検出漏れを減らしたい場合）
automosaic photo.jpg --confidence 0.3

# 感度を下げる（誤検出を減らしたい場合）
automosaic photo.jpg --confidence 0.8
```

### `--padding` / `-p` -- 検出領域の余白

検出された顔領域に追加する余白の割合を 0.0〜1.0 で指定します（デフォルト: 0.2）。

0.2 の場合、顔領域の幅・高さをそれぞれ 20% ずつ拡張します。

```bash
# 余白を広めに取る
automosaic photo.jpg --padding 0.4
```

### `--whitelist` / `-w` -- ホワイトリスト

モザイク処理から除外したい人物の参照画像が入ったディレクトリを指定します。

```bash
automosaic group_photo.jpg --whitelist ./my_face/
```

ホワイトリスト機能を使うには追加パッケージが必要です:

```bash
pip install "automosaic[whitelist]"
```

参照画像のディレクトリには、除外したい人物の正面顔写真を1枚以上入れてください。

### `--recursive` / `-r` -- 再帰処理

ディレクトリ指定時にサブディレクトリ内のファイルも処理します。

```bash
automosaic ./photos/ --recursive
```

### `--overwrite` -- 上書き

既に出力先に同名ファイルが存在する場合に上書きします。指定しない場合はスキップされます。

### `--gpu` -- GPU アクセラレーション

CUDA 対応 GPU が利用可能な場合に GPU を使用して高速化します。GPU が利用できない場合は警告を出して CPU にフォールバックします。

```bash
automosaic video.mp4 --detector yolo --gpu
```

### `--verbose` / `-v` -- 詳細ログ

DEBUG レベルのログを出力します。問題の調査時に使用します。

### `--dry-run` -- シミュレーション

実際の処理を行わず、処理対象ファイルの一覧と設定内容を表示します。

```bash
automosaic ./photos/ --recursive --dry-run
```

### `--version` / `-V` -- バージョン表示

```bash
automosaic --version
```

## 対応ファイルフォーマット

### 画像

| 拡張子 | 読み込み | 書き出し |
|--------|---------|---------|
| `.jpg` / `.jpeg` | 対応 | 対応 |
| `.png` | 対応 | 対応 |
| `.bmp` | 対応 | 対応 |
| `.webp` | 対応 | 対応 |

### 動画

| 拡張子 | 読み込み | 書き出し |
|--------|---------|---------|
| `.mp4` | 対応 | 対応 (H.264) |
| `.mov` | 対応 | 対応 (H.264) |
| `.avi` | 対応 | 対応 |
| `.mkv` | 対応 | 対応 (H.264) |

## エラーコードと対処法

| 終了コード | エラー種別 | 対処法 |
|-----------|-----------|--------|
| 1 | 入力ファイルが存在しない | ファイルパスを確認してください |
| 1 | 非対応フォーマット | 上記の対応フォーマット一覧を確認してください |
| 0 | 顔が検出されない | `--confidence` を下げるか、`--detector yolo` を試してください |
| 1 | 出力先に書き込み不可 | 出力先ディレクトリのパーミッションを確認してください |
| 1 | 動画コーデックエラー | ffmpeg がインストールされているか確認してください: `ffmpeg -version` |
| 0 | GPU が利用不可 | CUDA ドライバをインストールするか、`--gpu` を外して CPU で処理してください |
| 1 | メモリ不足 | 入力ファイルの解像度を下げるか、バッチサイズを調整してください |
| 0 | ホワイトリスト画像に顔なし | 参照画像に正面顔が写っているか確認してください |

## パフォーマンスガイドライン

### 処理時間の目安

| 処理対象 | 条件 | 想定処理時間 |
|---------|------|-------------|
| 画像 1枚 (1920x1080) | CPU (MediaPipe) | 500ms 以下 |
| 画像 1枚 (1920x1080) | GPU (YOLO) | 200ms 以下 |
| 動画 1分 (1080p/30fps) | CPU (MediaPipe) | 3分以下 |
| 動画 1分 (1080p/30fps) | GPU (YOLO) | 1分以下 |
| バッチ 100枚 | CPU (MediaPipe) | 60秒以下 |

上記はハードウェア構成によって変動します。

### パフォーマンス改善のヒント

1. **GPU を使う** -- CUDA 対応 GPU がある場合は `--gpu` オプションで大幅に高速化できます
2. **MediaPipe を使う** -- GPU がない場合、MediaPipe は YOLO より高速に動作します
3. **信頼度閾値を上げる** -- `--confidence` を高くすると、検出処理が軽くなります
4. **解像度を下げる** -- 入力ファイルの解像度が高い場合、事前に縮小すると処理が速くなります
5. **SSD/NVMe を使う** -- 大量のファイルをバッチ処理する場合、ストレージ速度が影響します
