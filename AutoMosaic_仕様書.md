## 1. 概要

### 1.1 プロジェクトの目的

AutoMosaic は、画像および動画ファイルに含まれる人物の顔を自動検出し、モザイク処理を施すコマンドラインツール（CLI）です。プライバシー保護やSNS投稿前の匿名化処理を、専門的な画像編集ソフトを使わずにワンコマンドで実行できることを目指します。

### 1.2 想定ユーザー

- ブログやSNSに写真・動画を投稿する個人ユーザー
- 取材映像・街頭インタビュー動画を編集するクリエイター
- 社内資料やプレゼンテーションで人物のプライバシーを保護したい企業ユーザー
- バッチ処理で大量の画像・動画を匿名化したい開発者

### 1.3 主な特徴

- ディープラーニングベースの高精度な顔検出（MediaPipe / YOLO 選択可能）
- 画像・動画の両方に対応（JPG, PNG, BMP, MP4, MOV, AVI, MKV）
- モザイク強度・スタイルのカスタマイズ（ピクセレート / ガウスぼかし）
- バッチ処理対応（ディレクトリ単位での一括処理）
- GPU アクセラレーション対応（CUDA）
- 特定の顔を除外するホワイトリスト機能

## 4. CLI インターフェース仕様

### 4.1 基本コマンド

**インストール:**

```bash
pip install automosaic
```

**基本的な使い方:**

```bash
automosaic <入力パス> [オプション]
```

### 4.2 コマンドオプション一覧

| オプション | 短縮形 | 型 | デフォルト | 説明 |
|-----------|--------|-----|-----------|------|
| `<INPUT>` | - | PATH | (必須) | 入力ファイルまたはディレクトリ |
| `--output` | `-o` | PATH | `./output/` | 出力先パス |
| `--style` | `-s` | STR | `pixelate` | モザイクスタイル (`pixelate` / `blur`) |
| `--strength` | `-S` | INT | `15` | モザイク強度 (1〜50) |
| `--detector` | `-d` | STR | `mediapipe` | 検出エンジン (`mediapipe` / `yolo`) |
| `--confidence` | `-c` | FLOAT | `0.5` | 検出信頼度の閾値 (0.0〜1.0) |
| `--padding` | `-p` | FLOAT | `0.2` | 検出領域の余白率 (0.0〜1.0) |
| `--whitelist` | `-w` | PATH | `None` | 除外する顔の参照画像ディレクトリ |
| `--recursive` | `-r` | FLAG | `False` | ディレクトリを再帰的に処理 |
| `--overwrite` | - | FLAG | `False` | 既存の出力ファイルを上書き |
| `--gpu` | - | FLAG | `False` | GPU アクセラレーションを有効化 |
| `--verbose` | `-v` | FLAG | `False` | 詳細ログの出力 |
| `--dry-run` | - | FLAG | `False` | 処理をシミュレーション（実行しない） |
| `--version` | `-V` | FLAG | - | バージョン情報の表示 |

### 4.3 使用例

**画像1枚にモザイク処理:**

```bash
automosaic photo.jpg
```

**動画にぼかしモザイクを適用:**

```bash
automosaic video.mp4 --style blur --strength 25
```

**ディレクトリ内を一括処理:**

```bash
automosaic ./photos/ -o ./output/ --recursive
```

**自分の顔を除外して処理:**

```bash
automosaic group_photo.jpg --whitelist ./my_face/
```

**高精度モードで処理:**

```bash
automosaic event.mp4 --detector yolo --gpu
```

## 5. 内部処理フロー

### 5.1 画像処理フロー

1. 入力ファイルを読み込み、対応形式かを判定
2. 選択された検出エンジンで顔領域を検出（バウンディングボックス取得）
3. ホワイトリストが指定されている場合、顔特徴量を比較し除外対象を判定
4. `padding` パラメータに基づき検出領域を拡張
5. 指定されたスタイル・強度でモザイク処理を適用
6. 出力先に保存（元のメタデータを可能な限り保持）

### 5.2 動画処理フロー

1. 動画ファイルを OpenCV でオープンし、メタデータ（FPS, 解像度, コーデック）を取得
2. フレーム単位で順次読み込み
3. 各フレームに対して画像処理フローと同様の顔検出・モザイク処理を実行
4. 処理済みフレームを出力動画に書き込み
5. 音声トラックを ffmpeg で元動画からコピー（音声は無加工）
6. 進捗状況を tqdm プログレスバーで表示

### 5.3 ホワイトリスト処理

ホワイトリスト機能は、指定された参照画像に含まれる顔の特徴量（face embedding）を事前に計算し、検出された各顔と照合します。類似度が閾値を超える場合、その顔はモザイク処理の対象外とします。この機能には `face_recognition` ライブラリ（dlib ベース）を使用します。

## 6. モジュール設計

### 6.1 detector モジュール

#### 抽象基底クラス（base.py）

```python
class FaceDetector(ABC):
    @abstractmethod
    def detect(self, frame: np.ndarray) -> list[BoundingBox]:
        """フレームから顔を検出しバウンディングボックスのリストを返す"""
        ...
```

`BoundingBox` は `NamedTuple` として定義し、`x`, `y`, `width`, `height`, `confidence` の各フィールドを持ちます。

#### MediaPipe 実装（mediapipe.py）

MediaPipe Face Detection を使用した軽量実装です。CPU 環境でも高速に動作し、デフォルトの検出エンジンとして採用します。

#### YOLO 実装（yolo.py）

YOLOv8-face モデルを使用した高精度実装です。GPU 環境での使用を推奨し、小さい顔や横顔の検出精度が向上します。オプション指定時のみロードされます。

### 6.2 mosaic モジュール

| スタイル名 | アルゴリズム | パラメータ |
|-----------|-------------|-----------|
| `pixelate` | 対象領域を縮小後に拡大（ニアレストネイバー） | `strength`: ブロックサイズ (px) |
| `blur` | ガウシアンぼかし | `strength`: カーネルサイズ |

### 6.3 processor モジュール

`ImageProcessor` と `VideoProcessor` は、共通の `Detector` と `MosaicStyle` を注入して使用する設計とし、テスタビリティを確保します。

### 6.4 utils モジュール

- **file_io.py:** 対応フォーマット判定、パス解決、出力先ディレクトリ作成
- **logger.py:** Python `logging` モジュールのラッパー。`--verbose` フラグで DEBUG レベルに切替

## 8. エラーハンドリング

| エラー種別 | 終了コード | 対応 |
|-----------|-----------|------|
| 入力ファイルが存在しない | 1 | エラーメッセージを表示して終了 |
| 非対応フォーマット | 1 | 対応フォーマット一覧を表示して終了 |
| 顔が検出されない | 0 | 警告を表示し、元ファイルをそのまま出力 |
| 出力先に書き込み不可 | 1 | パーミッションエラーを表示 |
| 動画コーデックエラー | 1 | ffmpeg のインストール案内を表示 |
| GPU が利用不可 | 0 | 警告を出して CPU フォールバック |
| メモリ不足 | 1 | 解像度縮小やバッチサイズ調整を提案 |
| ホワイトリスト画像に顔なし | 0 | 警告を表示して処理を継続 |

## 9. テスト計画

### 9.1 ユニットテスト

- 顔検出器の検出結果が正しいバウンディングボックス形式であること
- モザイク処理後の画像サイズが変わらないこと
- 各モザイクスタイルが正しく適用されること
- ホワイトリスト照合の一致・不一致判定が正しいこと
- 不正なパラメータに対して適切な例外が発生すること

### 9.2 統合テスト

- 画像ファイル（各フォーマット）のエンドツーエンド処理
- 動画ファイルの処理と音声保持の確認
- ディレクトリ一括処理（`--recursive` 含む）
- ホワイトリスト除外機能の動作確認

### 9.3 テストカバレッジ目標

コアモジュール（detector, mosaic, processor）のカバレッジ **80% 以上**を目標とします。
